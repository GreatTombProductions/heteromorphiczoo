## 1. The Register: Reverent-Irreverent

The PR context bundle specifies the register for press materials. The website register is adjacent but distinct — it's *environmental*, not *communicative*. Press copy talks about the band. Website copy IS the band's space. The visitor isn't being told about a cathedral. They're inside one.

**What the register IS:**
- Sacred text inscribed into the walls of something built to last
- Liturgical grammar: declarations, invocations, blessings. Sentences that feel carved, not typed
- The dry wit of something holy that knows it's also absurd. Not comedy — *confidence*
- Language that rewards the returning visitor. The words that confused you on your first visit are the words you use to describe this place on your third
- Warmth underneath the gravity. The menagerie is *welcomed*, not interrogated

**What the register is NOT:**
- Not the press register. Press copy is addressed outward (to journalists, to strangers). Website copy is addressed inward (to the space itself, to anyone who enters)
- Not a narrator describing the experience. The copy IS the experience. "You have been counted" is not narration — it's consecration

**The stained-glass correction:** The council defaulted to "dark cathedral" language. The Lordigan artwork corrected this — the palette is luminous, not dark. The copy register harmonizes: the language should feel *radiant*, not *austere*. A cathedral with light pouring through stained glass, not a crypt. The gravity is still there — but the gravity of something alive and glowing, not something buried and cold.

## 2. Copy Anti-Pattern List (Load-Bearing)

Every anti-pattern below is a training-distribution basin an implementation session will drift toward. The first items are the strongest attractors.

### Hard Prohibitions — Universal

| Anti-Pattern | Why it's wrong |
|-------------|---------------|
| "Welcome to our website" | Generic greeting. The site is a space, not a host. You don't welcome someone into a cathedral — they enter |
| "Click here to learn more" | Instructional web copy. The lowest-resolution text on the internet |
| "Submit your email for updates" | Transactional newsletter language. The menagerie isn't a mailing list |
| "Join thousands of fans" | Social proof by number. The Census provides this structurally. The copy doesn't need to advertise it |
| "Don't miss out" / "Limited time" | FOMO language. Explicitly banned in the engagement spec. The system rewards sustained devotion, not panic |
| "We're excited to announce" / "thrilled to share" | Corporate release language. Second-highest drift target after generic metal hype |
| "Level up!" / "Earn points!" / "Unlock rewards!" | Gamification language. The DP system IS gamification, but the text NEVER acknowledges it as such. Liturgical vocabulary, not ludic |
| "Check out our new..." | Casual web browsing language. The menagerie doesn't "check things out." They receive offerings |
| "Content" (as in "fan content") | The word "content" strips creative work of its character. Use "Offerings." Always |
| "Subscribe" / "Sign up" / "Register" | Transaction verbs. Use consecration verbs: "join," "enter," "be counted" |

### Hard Prohibitions — Metal-Specific

These come from the PR context bundle and apply to every surface, not just press materials:

| Anti-Pattern | Why |
|-------------|-----|
| "Crushing" / "brutal" / "insane" / "banger" | Generic metal hype. Highest-density training basin |
| "Dropping" (as in "track drops") | The track is not dropping. It arrives, descends, is bestowed, or simply: is out |
| Genre taxonomy lectures | Don't explain what melodic death metal is. Not here |
| "Just a small band" / "our little project" | Self-deprecating indie language. HZ is not asking permission |
| Excessive emoji in structural copy | Social captions can be warmer. Structural site copy is composed |

### Soft Prohibitions (Avoid Unless Serving a Specific Function)

| Pattern | When it might be acceptable |
|---------|---------------------------|
| Second person imperative ("Listen now") | Acceptable in CTA buttons, not in body copy |
| Numbers in prose ("over 200 fans") | Acceptable on the press page (industry consumer). Never on atmospheric pages |
| Hashtags | Acceptable in social share templates. Never in site body copy |
| Band member first names without context | Acceptable on the Bestiary page. Avoid in atmospheric copy where the band is a collective, not individuals |

## 3. Text Surface Register Catalog

Every text surface on the site, with its register, anti-patterns, and example copy. Implementation sessions use this as a lookup table.

### 3.1 Navigation Labels

**Register:** Liturgical-functional. The site sections are rooms in a cathedral, named for what they hold.

| Section | Label | NOT |
|---------|-------|-----|
| Home / Landing | — (no label; it's the entrance) | "Home" |
| Band members | The Bestiary | "Band" / "Members" / "About" |
| Fan art / UGC gallery | Offerings | "Fan Art" / "Gallery" / "Community" |
| YouTube reactions | Reactions | "React" / "Videos" |
| Fan leaderboard | Menagerie Roll | "Leaderboard" / "Community" |
| Events / challenges | The Rites | "Events" / "Challenges" |
| Band history | The Chronicle | "Timeline" / "History" |
| Press / EPK | Press | "Press" (this one is plain — intentional register break for industry) |
| Merch | Relics | "Shop" / "Merch" / "Store" |

**Note:** Navigation labels are rendered in the functional sans-serif per the Visual Design Spec. They're the one surface where the type is recessive. The labels do the liturgical work; the typeface stays quiet.

### 3.2 The Landing Page

**Register:** Atmospheric-invitational. The first text the visitor encounters. Minimal words, maximum weight.

**The hierarchy of the landing page text:**

1. **Band name** — "HETEROMORPHIC ZOO" — the logo, not typed text. No tagline underneath. The name is the tagline.

2. **Featured content callout** — whatever's currently featured (Benediction on launch). Short, declarative. Not a description — an announcement.
   - Example: "Benediction. The first blessing of the new order. Feat. Coty Garcia."
   - NOT: "Check out our new single Benediction, featuring founding vocalist Coty Garcia!"

3. **The email capture** — see Section 3.3

4. **A single atmospheric line** — optional, below the fold. Something that rewards the visitor who scrolls.
   - Example: "Worship music for monsters."
   - Or: "The menagerie assembles."
   - NOT: "Welcome to the official website of Heteromorphic Zoo"

**Anti-pattern for the landing page:** Too many words. The visual design carries the three-second experience. The copy exists to name things, not to explain them. Every word on the landing page should survive the question: "would this be carved into stone?"

### 3.3 Email Capture

**Register:** Invitational-inevitable. Not asking. Announcing your eligibility to be counted.

**The flow:**

| State | Copy |
|-------|------|
| **Prompt** | "Join the menagerie." |
| **Input placeholder** | "Your name for the roll" |
| **Button** | "Enter" |
| **Success** | "You have been counted." |
| **Already registered** | "You are already among us." |
| **Error (invalid email)** | "The roll requires a true name." |

**Anti-patterns:**
- "Subscribe to our newsletter" — transaction language
- "Get updates and exclusive content" — promise language. The menagerie doesn't promise. It consecrates
- "Thanks for subscribing!" — gratitude where there should be gravitas
- Multi-field forms. Single field. Email only. The engagement spec says 23% conversion for single-field vs. 8% for multi-field. The single field also matches the register: you give your name. That's the ritual. Anything more is paperwork

### 3.4 Confirmation Flows

**Register:** Consecration language. Every confirmation is a small ritual completion.

| Event | Copy |
|-------|------|
| **Email signup confirmed** | "You have been counted among the menagerie. The roll bears your name." |
| **Offering submitted** | "Your offering has been received. It awaits consecration." |
| **Offering approved** | "Your offering has been consecrated. It stands in the gallery." |
| **Offering rejected** | "The altar did not accept this offering. You may present another." |
| **Rank-up: Acolyte** | "You have been consecrated as Acolyte. The menagerie acknowledges your devotion." |
| **Rank-up: Deacon** | "Rise, Deacon. Your devotion is witnessed. The inner sanctum opens." |
| **Rank-up: Elder** | "The Elders receive you. Your name shall appear in the sacred texts." |
| **Rank-up: High Priest/Priestess** | "The highest offices recognize few. You are known to us now." |
| **Rank-up: Archbishop** | "Archbishop. The menagerie kneels. A relic shall bear your name." |

**Anti-patterns:**
- "Congratulations! You've reached Level 2!" — gamification language
- "Your submission is under review" — corporate moderation language. "Awaits consecration" says the same thing in the right register
- "Thanks for your submission!" — gratitude is fine. Generic gratitude is not. The confirmation should feel like something *happened*, not like a form was processed

### 3.5 Error States

**Register:** Profane-diagnostic. The liturgy breaks and something underneath shows through. Errors are the only surface where the register allows a crack — something rawer, less composed. Not angry. Wry.

| Error | Copy |
|-------|------|
| **General error** | "Something broke the ritual. Try again." |
| **Upload failed** | "The altar rejected your offering. The format was unworthy." |
| **Network error** | "The connection to the menagerie was severed. The faithful persist." |
| **Rate limited** | "Your devotion exceeds the altar's capacity. Wait, and return." |
| **Server error (500)** | "The cathedral shudders. The architects have been summoned." |
| **File too large** | "The offering exceeds the altar's measure. Compress and return." |
| **Invalid URL (reaction submission)** | "This does not lead where you claim. Provide a true path." |

**Anti-patterns:**
- "Oops! Something went wrong" — infantile error language
- "Error 500: Internal Server Error" — raw technical. The visitor never sees HTTP codes
- "Please try again later" — generic. The liturgical alternative tells you WHY while staying in register

### 3.6 The 404 Page

**Register:** This is a creative opportunity every band website wastes. The 404 is a room that shouldn't exist — the visitor has wandered beyond the known rites. The tone: not an error. A boundary.

**Copy:**
```
You have wandered beyond the known rites.

The menagerie does not extend this far.

[Return to the altar →]
```

**Anti-patterns:**
- "Page not found" — technically correct, atmospherically dead
- A generic 404 illustration (broken robot, lost astronaut)
- Redirect to homepage without acknowledgment

**Visual note:** The 404 page should use the deepest background values (`--hz-plum`) with minimal texture — the feeling of standing at the edge of the lit space, looking into the dark. The "Return to the altar" link in `--hz-glow-magenta` is the only warm light.

### 3.7 Loading States

**Register:** Atmospheric, not functional. Loading is the threshold between arriving and being inside.

| Context | Copy |
|---------|------|
| **Page loading** | "The congregation assembles..." |
| **Gallery loading** | "The offerings emerge..." |
| **Reactions wall loading** | "The witnesses gather..." |
| **Search results loading** | "The archives open..." |
| **Menagerie Roll loading** | "The roll unfurls..." |

**Anti-patterns:**
- Spinning wheel with no text
- "Loading..." — functional, not atmospheric
- Progress bars with percentages — this is a cathedral, not a download manager

**Implementation note:** Loading states should use the `--hz-bone-dim` color and the body serif typeface. They're quiet, not performative. The ellipsis is the only punctuation that belongs in loading states — it implies continuation, not completion.

### 3.8 Empty States

**Register:** When a gallery or section has no content yet, the emptiness is an invitation, not an error.

| Context | Copy |
|---------|------|
| **Offerings gallery (empty category)** | "No offerings in this rite. The altar waits." |
| **Reactions wall (empty)** | "No witnesses yet. Be the first to testify." |
| **Search with no results** | "The archives hold nothing by that name." |
| **Menagerie Roll (no members in a tier)** | "None have risen this far. The rank awaits its first." |

**Anti-patterns:**
- "No results found. Try a different search." — generic web
- "Be the first to contribute!" — startup community language
- Illustration of an empty box or folder

### 3.9 Footer / Legal

**Register:** Even the legal text has the register. The register never breaks. The footer is the last thing a visitor reads before leaving — it should feel like the inscription at the base of a monument, not the fine print on a contract.

**Copyright:** `© MMXXVI Heteromorphic Zoo. All rites reserved.`

**Note:** "All rites reserved" — not "rights." This is intentional. It's the smallest possible joke that also happens to be legally functional.

**Links in footer:** Privacy policy, terms — these can be plain. The legal documents themselves don't need the register (they need to be legally valid). But the footer labels can: "The Law" instead of "Terms of Service." "Privacy Rites" instead of "Privacy Policy."

### 3.10 The Bestiary (Band Members Page)

**Register:** Each band member is rendered as a creature in the menagerie. The descriptions are reverent but specific — real musicianship described through the zoo metaphor.

**Structure per member:**
- **Name** (display type)
- **Designation** (what they are in the zoo)
- **Description** (2-3 sentences, liturgical register, but grounded in what they actually do)

**Example entries:**

**Ray Heberer — The First Disciple**
*Composition, arrangement, guitars, vocals, production.*
Every note in the zoo is written by this hand. The arrangements are scripture — violin, guitar, and orchestral voices interwoven from inception, not layered afterward. A decade of metal projects before the zoo existed. The one who heard the sound first and transmits it. Now the voice as well as the architect.

**Megan Ash — The Siren**
*Violin.*
The violin is structural, not decorative. Written into the DNA of every composition — a three-voice fugue with the lead guitar and rhythm section in "Ritual of Fidelity." Her solo career launched with "Hexed" (2025). The siren's call is not embellishment. It is load-bearing.

**Bryce Butler — The Engine**
*Drums, drum arrangement.*
The rhythmic foundation the cathedral is built upon. From the bone-moving bass-and-drum duet that opens "Your Final Seconds" to the jackhammer precision that drives "Napalm."

**Coty Garcia — The Founding Voice**
*Vocals (founding member, EP era).*
The voice that first spoke the rites. Every track on the New World EP, both preceding singles — "utterly deranged vocals that sound like a fight between bull elephants, howler monkeys, and a pack of demons that just escaped Hell." Honored on "Benediction" — a dual-vocalist blessing where the founding voice and the new speak together. Not every band says goodbye with a song.

**Anti-patterns:**
- Listing instruments and influences like a resume
- "Meet the band!" heading language
- Stock band photo grid
- Omitting Coty or relegating Coty to a footnote

**Anti-patterns for the timeline:**
- Bulleted list of dates and facts
- "2024 — Released Napalm single" — resume format
- Neutral, journalistic tone. The timeline is partisan — it tells the story from inside the zoo
- Omitting the prehistory. The 13-year arc from Taiwan to British Columbia IS the story

### 3.12 The Press Page (/press)

**Register break — intentional.** This page serves industry consumers (labels, booking agents, press, playlist curators). The register shifts to professional, clean, numbers-forward. The liturgical vocabulary is dialed back. Industry people want to see the cult working. They don't want to be in it.

**Structure:**
- **One-liner:** "Heteromorphic Zoo — worship music for monsters."
- **Short bio** (50 words, from PR context bundle)
- **Census stats** (aggregate numbers — total menagerie, tier distribution)
- **Press highlights** (NCS, Circle Pit, FemMetal Rocks quotes with links)
- **Production credits** (EP: Chris Wiseman, Christian Donaldson; Benediction: Ray Heberer, Greg Thomas, Joel Wanasek; Guest: Francesco Ferrini — names that carry weight)
- **High-res press photos** (download links)
- **Booking / press contact**
- **EPK download**

**The register here:** Professional, warm, concise. Not liturgical. Not generic. The press page is a handshake — firm, direct, aware that the person reading it has 200 other bands to evaluate today. The tagline survives ("worship music for monsters") because it IS the pitch. The liturgical vocabulary does not survive — "Devotion Points" means nothing to an A&R person. "847 active fans creating unprompted content" does.

**Anti-patterns:**
- "Welcome to our press page" — nobody says this
- Embedding the full liturgical vocabulary without translation
- Lengthy bio that reads like a Wikipedia article
- No numbers. The press page exists FOR numbers. Census, streams, press coverage count, UGC volume — industry reads numbers

## 4. Benediction-Specific Copy

The Monday launch needs specific copy for the Benediction single page. This is the page the Instagram CTA drives to — the resolution of the lineup change theater.

### The Benediction Single Page

**Headline:** "Benediction"
**Subheadline:** "feat. Coty Garcia"

**Body copy (3-4 lines, max):**

"A blessing spoken in two voices. The founding voice and the new. Written in the early days, completed in the last days of the old order.

Not every band says goodbye with a song.

Benediction is the first rite of the new era."

**CTA block:**
- Pre-save / stream button: "Receive the blessing"
- Music video link (when live): "Witness"

**Credits block (smaller text, `--hz-bone-dim`):**

"Mixed by Ray Heberer · Produced by Greg Thomas · Mastered by Joel Wanasek · Video by Scott Hansen."

**Anti-patterns for the Benediction page:**
- Explaining the lineup change in prose. The copy implies it. The music embodies it. Don't narrate what the listener will hear
- "We're excited to share..." — the Benediction page is not excited. It is composed. It is a benediction
- Genre descriptions. Nobody needs to be told what kind of metal this is while they're listening to it
- "Featuring Coty Garcia" without context. "Feat. Coty Garcia" is the standard musical credit format and it's sufficient. The Bestiary and Chronicle tell the full story

## 5. Microcopy Patterns

These are the small text elements that appear throughout the site. They're the mortar between the stones — individually invisible, collectively they determine whether the building holds.

### Date Formatting

Dates on the site should feel historical, not digital.

- Full dates: "October 10, 2024" — not "10/10/24" or "2024-10-10"
- Year-only: "MMXXIV" in display contexts (the copyright, the timeline section headers). Standard numerals in data-dense contexts (the press page, the Reactions Wall card metadata)
- Relative dates: never. No "3 days ago" or "just now." Every date is a fixed point in the chronicle

### Rank Display

When a fan's rank appears next to their name (in the Roll, on Offering cards, in confirmation flows):

- Format: `[Name] · [Rank]`
- Example: `Sarah Kim · Deacon`
- The rank name is always capitalized. It's a title, not a label
- At Elder and above, the rank is displayed in `--hz-gold`

### Button Text Patterns

| Action Type | Pattern | Example |
|------------|---------|---------|
| Primary CTA | Imperative, single word or two | "Enter" / "Offer" / "Present" |
| Navigation | Destination name | "The Bestiary" / "Offerings" |
| External link | Action + destination | "Stream on Spotify" / "Watch on YouTube" |
| Return / back | Liturgical return | "Return to the altar" / "Back to the rites" |

**Anti-patterns:** "Click here" / "Learn more" / "Submit" (too generic) / "Go!" (too casual)

### Tooltip / Helper Text

When a UI element needs explanation (the DP display, the rank system, submission requirements):

- Register: instructional but warm. A guide, not a manual
- Example: "Devotion Points measure the depth of your participation. Not streams — devotion."
- NOT: "Devotion Points (DP) are earned by performing eligible engagement actions as detailed in our terms."

## 6. The Three Consumers — Voice Calibration

From Shihoutu's two-consumer model (extended by Rimuru to three):

### Consumer 1: The Uninitiated

The liturgical vocabulary is *strange* to them. "Offerings." "The Menagerie." "Consecration." They don't know what these words mean in this context. That strangeness IS the three-second experience extended through language. The visitor doesn't understand the vocabulary, and that incomprehension draws them deeper — "what IS this place?"

**Voice calibration:** Never explain the vocabulary to the uninitiated on atmospheric pages. Let the strangeness do its work. The visitor learns by inhabiting the space, not by reading a glossary. The words will make sense on their third visit. That's the retention mechanism — the language becomes *inside language* through repeated exposure.

### Consumer 2: The Returning Menagerie

The same vocabulary that confused a stranger is now home. "Offerings" isn't weird anymore — it's what they call their fan art. "Consecration" isn't liturgical theater — it's what happens when their work gets approved.

**Voice calibration:** Consistency is everything. The register NEVER breaks — not in error states, not in loading states, not in the footer, not anywhere. A single "Upload failed" instead of "The altar rejected your offering" shatters the world. The returning member trusts the register. Breaking it breaks trust.

### Consumer 3: Industry

Labels, booking agents, press outlets, playlist curators. They want numbers, EPK, booking contact.

**Voice calibration:** The /press page breaks register intentionally. Professional, clean, numbers-forward. "847 active fans creating unprompted content" — not "847 Acolytes consecrated into the menagerie." Industry people don't want to feel recruited. They want to evaluate the cult from outside. The tagline survives ("worship music for monsters") because it IS the pitch. Everything else translates to industry language.

- **Social media captions** — handled by the PR context bundle's format-specific guidance
- **Email marketing / Dittofeed journeys** — future phase; the register principles here apply, but the specific copy for automated email sequences should be specced when that system is built

- **Blog / news posts** — if the site has a blog or news section (not currently in the campaign), the register should be a less formal variant of the liturgical voice — the band speaking directly, not the cathedral's inscriptions

## 8. Implementation Notes for Rubedo

1. **Every string is in a constants file.** Don't hardcode copy into components. A single `copy.ts` or `content.ts` file with all text strings, organized by surface. This makes future copy changes trivial — edit one file, not hunt through 40 components.

2. **The register must survive component reuse.** If a generic `<Button>` component is used across surfaces, the text it displays must still be surface-specific. Don't let component abstraction flatten the register.

3. **Test the register by reading the page aloud.** If any sentence sounds like it could appear on a SaaS landing page, a startup homepage, or a generic dark-mode website, it's wrong. The copy should sound like it was written by a devotee, not a designer.

4. **Empty states and error states are first-class copy.** Don't default to browser/framework defaults and "fix them later." The error state copy in this spec is load-bearing — it's where the register either holds or breaks. Implement them from the start.

5. **The press page is the ONE exception.** Every other page is liturgical. The press page is professional. Don't let the liturgical register bleed into the press page, and don't let the press page's plainness bleed back.
